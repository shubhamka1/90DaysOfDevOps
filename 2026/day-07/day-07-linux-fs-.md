# Part 1: Linux File System Hierarchy

Linux follows a hierarchical directory structure where every file and directory starts from the root directory `/`. Understanding the purpose of important directories is essential for system administration, troubleshooting, automation, and DevOps work.

---

# 1. / (Root Directory)

## Purpose

The root directory is the top-most directory in Linux. Every file and directory on the system exists under `/`.

## Common Contents

```bash
ls -l /
```

Typical output contains:

```text
bin
boot
dev
etc
home
lib
media
mnt
opt
root
run
srv
tmp
usr
var
```

## Important Notes

* Everything starts from `/`
* Similar to `C:\` in Windows
* Mount points for disks and filesystems are attached somewhere under `/`
* Most troubleshooting begins by identifying where files are located under the root filesystem

---

# 2. /home

## Purpose

Contains personal directories for regular users.

Example:

```text
/home/shubham
/home/oracle
/home/appuser
```

## Common Contents

```bash
ls -la /home/username
```

Important files:

### .bashrc

```bash
cat ~/.bashrc
```

Shell configuration file executed whenever a terminal session starts.

Common uses:

* Aliases
* Environment variables
* PATH settings

---

### .profile

User login configuration.

Executed during login.

---

### .ssh/

Contains SSH-related files.

Important files:

```text
authorized_keys
id_rsa
id_rsa.pub
known_hosts
```

#### authorized_keys

Controls who can SSH into the server.

#### id_rsa

Private SSH key.

#### id_rsa.pub

Public SSH key.

#### known_hosts

Stores fingerprints of remote systems.

## DevOps Importance

* SSH authentication
* Automation scripts
* User environment variables
* Service account configurations

---

# 3. /root

## Purpose

Home directory of the root user.

Different from `/`.

```text
/root
```

belongs only to root.

## Common Contents

```bash
ls -la /root
```

Contains:

```text
.bashrc
.ssh
scripts
backups
```

## DevOps Importance

Administrators often store:

* Troubleshooting scripts
* Backup scripts
* Emergency recovery files
* SSH keys

---

# 4. /etc

## Purpose

Contains system-wide configuration files.

One of the most important directories in Linux.

## Common Contents

```bash
ls -l /etc
```

---

## Important Files

### /etc/hostname

```bash
cat /etc/hostname
```

Stores system hostname.

Example:

```text
dw-prod-server01
```

---

### /etc/hosts

```bash
cat /etc/hosts
```

Local hostname resolution.

Example:

```text
127.0.0.1 localhost
```

Useful when DNS is unavailable.

---

### /etc/passwd

```bash
cat /etc/passwd
```

Stores user account information.

Example:

```text
oracle:x:1001:1001::/home/oracle:/bin/bash
```

Contains:

* Username
* User ID
* Group ID
* Home directory
* Default shell

---

### /etc/group

```bash
cat /etc/group
```

Stores group definitions.

Used to manage permissions.

---

### /etc/shadow

```bash
sudo cat /etc/shadow
```

Stores encrypted passwords.

Accessible only by root.

---

### /etc/fstab

```bash
cat /etc/fstab
```

Defines filesystem mount configuration.

Critical after server reboot.

Example:

```text
UUID=xxxx /data ext4 defaults 0 2
```

---

### /etc/crontab

```bash
cat /etc/crontab
```

Defines scheduled jobs.

Important for automation.

---

### /etc/resolv.conf

```bash
cat /etc/resolv.conf
```

DNS configuration.

Example:

```text
nameserver 8.8.8.8
```

---

### /etc/ssh/sshd_config

```bash
cat /etc/ssh/sshd_config
```

SSH server configuration.

Controls:

* Port number
* Root login
* Authentication methods

## DevOps Importance

Most production issues related to:

* Users
* DNS
* Mounts
* SSH
* Scheduling

are investigated from `/etc`.

---

# 5. /var/log

## Purpose

Stores logs generated by the operating system and applications.

Most important directory for troubleshooting.

## Common Contents

```bash
ls -l /var/log
```

---

## Important Log Files

### syslog

```bash
tail -50 /var/log/syslog
```

General system messages.

Useful for:

* Service failures
* Application messages
* System events

---

### auth.log

```bash
tail -50 /var/log/auth.log
```

Authentication logs.

Contains:

* SSH logins
* Sudo activity
* Failed logins

---

### kern.log

Kernel-related messages.

Useful for hardware troubleshooting.

---

### boot.log

Messages generated during boot.

---

### dmesg

```bash
dmesg | tail
```

Kernel ring buffer.

Shows:

* Hardware detection
* Disk errors
* Driver issues

---

### journal

Used by systemd.

Viewed using:

```bash
journalctl
```

Examples:

```bash
journalctl -u ssh
journalctl -xe
```

---

## Finding Large Logs

```bash
du -sh /var/log/* 2>/dev/null | sort -h | tail -5
```

Useful when disk space is running low.

## DevOps Importance

First location checked during incidents.

---

# 6. /tmp

## Purpose

Temporary storage.

Applications use it for temporary data.

## Common Contents

```bash
ls -l /tmp
```

Examples:

```text
temp files
installer files
application caches
```

## Important Notes

* Usually cleared after reboot
* Any user can write here
* Often used for testing scripts

---

# 7. /bin

## Purpose

Contains essential system commands required during boot and recovery.

## Common Commands

```text
ls
cp
mv
cat
chmod
chown
mkdir
rm
```

Verify:

```bash
ls /bin
```

These commands are available even in recovery mode.

---

# 8. /usr/bin

## Purpose

Contains most user-level commands and utilities.

## Common Commands

```text
grep
find
awk
sed
ssh
scp
curl
wget
vim
```

Verify:

```bash
ls /usr/bin | head
```

## DevOps Importance

Most troubleshooting commands are stored here.

# 9. /opt

## Purpose

Stores third-party software not installed as part of the operating system.

## Examples

```text
/opt/informatica
/opt/controlm
/opt/oracle
/opt/tomcat
```

## DevOps Importance

Enterprise applications are commonly installed here.

# Part 2: Scenario Practice

## Scenario 1: Application Not Responding

### Check Process

```bash
ps -ef | grep application
```

Verify application process exists.

---

### Check Service Status

```bash
systemctl status application
```

Verify service is active.

---

### Check Listening Ports

```bash
ss -tulpn
```

Verify application port is open.

---

### Review Logs

```bash
journalctl -u application
```

or

```bash
tail -100 /var/log/application.log
```

Look for:

* Errors
* Exceptions
* Connection failures

---

## Scenario 2: Disk Space Alert

### Check Filesystem Usage

```bash
df -h
```

Identify full filesystems.

---

### Find Large Directories

```bash
du -sh /* 2>/dev/null | sort -hr | head
```

Locate major space consumers.

---

### Investigate Logs

```bash
du -sh /var/log/*
```

Determine whether logs are causing disk growth.

---

## Scenario 3: User Cannot SSH

### Verify User

```bash
grep username /etc/passwd
```

Confirm account exists.

---

### Verify SSH Service

```bash
systemctl status ssh
```

---

### Review Authentication Logs

```bash
tail -50 /var/log/auth.log
```

Look for:

```text
Failed password
Account locked
Permission denied
```

---

### Check SSH Configuration

```bash
cat /etc/ssh/sshd_config
```

Verify authentication settings.

# Key Takeaways

* `/etc` contains configuration.
* `/var/log` contains logs.
* `/home` and `/root` contain user data.
* `/tmp` contains temporary files.
* `/bin` and `/usr/bin` contain commands.
* `/opt` contains enterprise applications.
* Most troubleshooting involves checking **processes, configurations, logs, disk space, network ports, and permissions**.
* For DevOps and production support roles, `/etc`, `/var/log`, `/home`, `/tmp`, and `/opt` are the directories accessed most frequently.
